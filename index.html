Our Mission? To provide a complete, Java-based solution for accessing, manipulating, and outputting XML data from Java code. Read more about our project goals.
Project Status? JDOM 1.1.3 is available! This is a recommended upgrade for anyone still using JDOM 1.1.2 or earlier.

This release fixes a single bug in 1.1.2 - SAXOutputter triggered a Namespace declaration for Attributes in Namespace.NO_NAMESPACE See Issue #60.
This release also includes the jdom-contrib artifact in maven-central.

The previous release (1.1.2) includes a number of minor bug fixes - See the issues fixed in JDOM 1.1.2.

JDOM 1.x versions are operating in maintenance mode only and only bug fixes will be considered for future 1.x releases. New features are being considered for JDOM2 though. If you want to be involved, or have suggestions or questions, please write in to the mailing list and check out the GitHub Repository.

The Latest?

Mission accomplished. We've succeeded in our mission with an API that's been through years of design and development and a codebase that is rock solid.

What about JSR-102? That's a good question. It hasn't been a priority for us. We seem to be doing pretty well on our own, it turns out.

If you wonder what Jason has been doing lately, he's working on MarkMail. It's a searchable email archive system. You can try searching the JDOM lists at jdom.markmail.org.
What's Next?

JDOM 2.0 is in BETA testing, and is close to being released. If you have time available for testing JDOM 2.0, or have comments, questions, or suggestions for something that is, or should be, in a 2.0 release, please write in to jdom-interest.
